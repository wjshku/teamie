Teamie 项目管理工具

用户操作流程与系统响应流程图

产品定位

为科研创业者设计的高效项目管理软件。帮助研究生/研究员在一个界面中快速了解各项目的整体进展，并通过周进展分析报告实现高效决策与复盘。

核心功能

  • F1 导入 Notion 项目内容：用户上传 Notion HTML 导出文件，系统自动读取会议记录、任务、文档等信息
  • F2 信息提取与分析：系统自动解析页面层级，提取关键要素（标题、摘要、日期、标签、任务状态）
  • F3 周进展分析报告：系统生成六个维度的结构化报告（动机与方向、达成事项、未达成事项、内部反思、外部反馈、下一步计划）

核心流程

用户导入 Notion HTML → 系统解析信息 → 对比上周"下一步计划" → 生成六个维度的周进展分析报告（动机与方向、达成事项、未达成事项、内部反思、外部反馈、下一步计划）→ 存储报告供查看/对比

关键功能映射

F1 - 导入 Notion 项目内容
触发：用户选择"导入项目"
执行：上传 HTML → 文件验证 → 解析完成
价值：快速接入现有项目资料，无需手动输入
F2 - 信息提取与分析
触发：文件解析成功
执行：识别层级 → 提取字段 → 分析数据
价值：将零散页面转化为结构化信息
F3 - 周进展分析报告
触发：信息提取完成
执行：分析内容 → 整理报告 → 生成输出
价值：帮助用户快速恢复上下文，便于复盘和决策
初次导入 vs 二次导入
初次导入：无历史数据 → 直接生成第一周报告 → 作为基准
二次导入：读取上周报告 → 对比上周"下一步计划" → 分析任务完成情况 → 生成六个维度的进展报告
六个维度进展分析
1. 动机与方向：为什么这样做
2. 达成事项：本周实际完成了什么
3. 未达成事项：计划要做但没做的事情
4. 内部反思：没做的原因分析
5. 外部反馈：成效和用户反馈
6. 下一步计划：下周的具体行动计划
